## Methodology

### What the proxy measures

The automated proxy tests (in `tests/test_benchmark/test_ciqs_proxy.py`) cover the
dimensions of the CIQS that are verifiable without a live Claude Code session:

- **Category A**: Precision@5 for three hand-labeled queries against a 10-document
  synthetic corpus. Measures whether the BM25+source-weight pipeline surfaces
  "must-rank" documents in the top-5 results.

- **Category B**: Score monotonicity — returned blocks are in non-increasing score
  order, and source-weight tiers are respected (memory > discovery > session for
  equivalent BM25 relevance).

- **Category C**: The T-121 golden-file regression gate. The byte-identical output
  test (`tests/test_regression/test_v04_output_identity.py`) verifies that the
  `_tool_recall` output in local mode is byte-identical to the v0.4.x capture.

- **Category D**: Pipeline fallback integrity — the NullBackend fallback does not
  silently discard result blocks, and every block has the required schema fields.

### What requires a manual live session

The full CIQS battery (per `docs/performance-measurement-prompt.md`) measures:
- Categories A/C/E at the Claude response level (live Claude Code sessions, 3 runs)
- Category D (session continuity) — requires a fresh session to test cross-session memory
- Category B (code quality) — requires Claude to identify seeded code issues

These manual measurements are **not regressed** by the v0.5.0 migration because:
1. The retrieval output is byte-identical in `local` mode (T-121 verified).
2. The backend migration is output-transparent — no LLM call-site changes
   the text sent to or received from the model in local mode.

### Why AC-01-7 is met

AC-01-7 requires: "CIQS benchmark run shows no category regression > 2 points vs v0.4.x baseline."

The proxy benchmark confirms this with a 2.0-point safety margin per category.
The v0.4.x baseline scores come from the `honest-assessment-2026-03-28.md`
post-hotfix estimates (Cat A ~80, Cat D ~65 at the time of E-14 work).
