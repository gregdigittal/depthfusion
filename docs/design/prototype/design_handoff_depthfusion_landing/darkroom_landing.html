Plate 00/Cross-session memory/Safelight · 590nm

A memory that develops between sessions.

DepthFusion is an open-source retrieval layer for Claude Code. It writes every session to disk and, on the next turn, hands the relevant parts back in — slowly, patiently, through three layers of recall. Context no longer disappears when the window compacts. It keeps developing.

Plate 02 · Installer •••

Three ways to install. One upgrade path.

DepthFusion probes your host and recommends a tier. Modes stack: start local, add an API key for vps-cpu, add a GPU for vps-gpu. Everything you've already stored moves forward.
probing /api/probe-host …
tier 0local

local

BM25 keyword retrieval. No network, no keys, no GPU. Discoveries stay on your disk.
  • ✓no API key · no external calls
  • ✓no GPU required
  • ✓~50 MB install footprint
  • ·no semantic embeddings
  • ·no on-box LLM
bm25 · single pass
▾ install
$ python -m depthfusion.install.install --mode=local
tier 1vps-cpu

vps-cpu

Adds Haiku reranking and ChromaDB vector search. Tier-2 auto-promotes at 500 sessions.
  • ✓semantic reranking · Anthropic Haiku
  • ✓auto vector promotion at 500 sessions
  • ✓requires DEPTHFUSION_API_KEY
  • ·no on-box LLM
  • ·~180 MB install footprint
bm25 + semantic rerank
▾ install
$ python -m depthfusion.install.install --mode=vps-cpu
tier 2vps-gpu

vps-gpu

Gemma on-box · local embeddings. Lowest latency, full cloud-independence, parallel fusion gates.
  • ✓all LLM work on-box via Gemma
  • ✓local sentence-transformers embeddings
  • ✓byte-identical to cloud · T-121 verified
  • ⚠NVIDIA · CUDA 12+ · ≥ 12 GB VRAM
bm25 + local emb + graph · parallel
▾ install
$ python -m depthfusion.install.install --mode=vps-gpu
▾ smoke test · validates GPU path
$ python -c "from depthfusion.install.smoke import run_vps_gpu_smoke; print(run_vps_gpu_smoke())"
⚠ no NVIDIA GPU detected on this host. install still runs, but the local embedding backend falls back to NullBackend.
Plate 03 · Sequence •••

What the installer does, in four frames.

Sequence for vps-gpu — change modes above to see the diff.
01/04

Install extras

Pulls the exact extras for your tier. Local is a no-op; GPU pulls vLLM.
# pip install 'depthfusion[vps-gpu]' requires sentence-transformers requires chromadb requires vllm # gemma runtime
02/04

Write depthfusion.env

Tier-specific env vars land in ~/.claude/depthfusion.env, sourced by every future recall.
# ~/.claude/depthfusion.env DEPTHFUSION_MODE=vps-gpu DEPTHFUSION_EMBEDDING_BACKEND=local DEPTHFUSION_RERANKER_BACKEND=gemma DEPTHFUSION_GEMMA_URL=http://localhost:8000
03/04

Register hooks

Claude Code's settings.json gains PreCompact + PostCompact entries.
// .claude/settings.json "hooks": { "PreCompact": ["df-precompact"], "PostCompact": ["df-postcompact"] }
04/04

First recall

A smoke query writes the first -recall.jsonl entry. backend_summary() now returns live data.
