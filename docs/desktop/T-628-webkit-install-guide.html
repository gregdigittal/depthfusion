Install webkit2gtk & gdk-pixbuf on the VPS

T-628 Lane C — Tauri 2 Shell VPS: 176.9.147.206  ·  Ubuntu 22.04 LTS

!

What's blocking the build — and why

The DepthFusion desktop app is built with Tauri 2, a framework that wraps your web UI inside a native desktop window. On Linux, Tauri uses the system's browser engine (WebKitGTK 4.1) to render that window.

The code is complete and correct. The build fails because the server is missing the development headers that the Rust compiler needs to link against WebKitGTK. Think of it like trying to build a LEGO set when the instruction booklet is missing — the pieces are fine; you just can't proceed without the reference.

⚠️

Tauri 2 requires libwebkit2gtk-4.1-dev, not the older libwebkit2gtk-4.0-dev. Installing the wrong version will appear to succeed but then fail at link time with cryptic symbol errors. This guide installs the correct version.

Update the package list

This refreshes the list of available packages from Ubuntu's servers. It doesn't install anything yet — it just makes sure apt knows about the latest versions.

server — takes ~10 seconds

You'll be asked for your password. Type it and press Enter. Nothing will appear as you type — that's normal on Linux.

Success looks like (last few lines)
...

If you see a lock error like this
E: Could not get lock /var/lib/dpkg/lock-frontend...

Another process is using apt. Wait 60 seconds and try again. If it persists:

server (only if lock error)
sudo killall apt apt-get 2>/dev/null; sudo rm -f /var/lib/dpkg/lock-frontend /var/lib/dpkg/lock; sudo dpkg --configure -a

Then retry sudo apt-get update.

3

Install the Tauri 2 build dependencies

This is the main step. It installs 7 packages that Tauri 2 requires on Linux. Copy and paste the entire block — it's one command spread across multiple lines using \ (backslash) for readability.

server — takes 1–3 minutes depending on bandwidth
sudo apt-get install -y \
  libwebkit2gtk-4.1-dev \
  libgdk-pixbuf-2.0-dev \
  libxdo-dev \
  libayatana-appindicator3-dev \
  librsvg2-dev \
  libssl-dev \
  build-essential

What each package does

PackageWhy Tauri 2 needs it
libwebkit2gtk-4.1-devThe browser engine (WebKit) that renders the app's UI inside the native window. Must be 4.1 — not 4.0.
libgdk-pixbuf-2.0-devImage rendering support. Used by GTK (the window toolkit) to display icons and images.
libxdo-devKeyboard/window automation library that Tauri links against on Linux.
libayatana-appindicator3-devSystem tray icon support. Required even if you're not using a tray icon.
librsvg2-devSVG rendering, used for the app's icon in various resolutions.
libssl-devTLS/HTTPS support for Tauri's HTTP client.
build-essentialThe C compiler (gcc) and linker (ld). Required for the Rust build to compile C dependencies.

Setting up libgdk-pixbuf-2.0-dev:amd64 (2.42.x+dfsg-1) ...

If you see: E: Unable to locate package libwebkit2gtk-4.1-dev

The Ubuntu package repositories might be from Ubuntu 20.04 which doesn't have 4.1. Add the PPA and retry:

server (only if package not found)
sudo add-apt-repository ppa:webkit-team/ppa -y && sudo apt-get update
sudo apt-get install -y libwebkit2gtk-4.1-dev libgdk-pixbuf-2.0-dev
If you see: E: Unable to locate package libayatana-appindicator3-dev

This package has a different name on some Ubuntu builds. Try the libappindicator variant:

server (only if libayatana-... not found)
sudo apt-get install -y libappindicator3-dev

✓ gdk-pixbuf-2.0 OK
If pkg-config says "No package ... found" even after installing

Refresh the shared library cache and retry:

server

Then re-run the two pkg-config commands above.

Install JavaScript dependencies

The frontend (TypeScript/React) needs its packages downloaded first. This is equivalent to pip install -r requirements.txt — it downloads packages into node_modules/.

npm install
Success
added 312 packages, and audited 313 packages in 47s

The exact package count doesn't matter. No npm error lines is what matters.

7

Verify the Rust code compiles

This is the key verification step. cargo check type-checks all the Rust code and confirms the linker can find the system libraries we just installed. It does not produce a final binary — it just validates everything compiles correctly. The first run downloads Rust crates (packages) and takes 2–4 minutes.

server — first run takes 2–4 minutes
cargo check --manifest-path src-tauri/Cargo.toml

What you'll see while it runs

Lines like these appear as Rust downloads and compiles dependencies. This is normal:

Normal output during run

Yellow warning: lines are fine — only red error[E...] lines are problems.

✅ SUCCESS — last line
    Finished `dev` profile [unoptimized + debuginfo] target(s) in 2m 34s
❌ FAIL — pkg-config error (environment problem, not code)
error: failed to run custom build command for `webkit2gtk v0.20.x`
...
pkg-config could not find: webkit2gtk-4.1

Go back to Step 4 — the package wasn't installed or ldconfig wasn't run.

❌ FAIL — Rust compile error (code problem, not environment)
error[E0412]: cannot find type `SomeType` in this scope

This is a code error, not an environment issue. Copy the full error block (everything between error[ and Finished) and send it back for diagnosis.
