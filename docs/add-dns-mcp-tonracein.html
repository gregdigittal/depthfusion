Add DNS record: mcp.tonracein.com

One A record in Cloudflare — takes about 60 seconds.

TypeA
Namemcp
IPv4 address176.9.147.206
TTLAuto
Proxy statusDNS only (grey cloud ☁️)
1

Open Cloudflare DNS for tonracein.com

Go to dash.cloudflare.com → click tonracein.com → click DNS in the left sidebar → click Records.

2

Click "Add record"

Blue button near the top of the DNS records table.

3

Fill in the form exactly as shown above

• Type: A
• Name: mcp
• IPv4 address: 176.9.147.206
• TTL: Auto
• Proxy status: click the orange cloud until it turns grey ("DNS only")

⚠️ Keep the cloud grey (DNS only). Orange/proxied breaks the Let's Encrypt certificate that Caddy auto-provisions — the server can't complete the ACME challenge through Cloudflare's proxy.
4

Click "Save"

That's it. DNS propagates in under a minute. Caddy will provision the TLS cert automatically on the first HTTPS request to mcp.tonracein.com.

Verify it worked

Run this from your Mac terminal a minute after saving:

dig A mcp.tonracein.com +short
# should return: 176.9.147.206

Then test the HTTPS endpoint (may take 30–60 s for Caddy to get the cert):

curl -s https://mcp.tonracein.com/health
# should return: {"status":"ok","transport":"sse","version":"2.0.0"}
✅ Once that curl returns 200, update the app's default server URL from http://176.9.147.206:7301 to https://mcp.tonracein.com in app/src/SettingsPage.tsx and app/src-tauri/src/settings.rs.
