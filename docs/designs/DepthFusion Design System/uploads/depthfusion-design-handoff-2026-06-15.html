Watermark is above the content layer (z-10) but pointer-events: none so text is selectable.

Design Note — Opacity Calibration At 8% opacity, the RESTRICTED text is barely visible against gray-900 surfaces. On lighter document content it will be more visible. Consider making opacity context-dependent (e.g., 0.06 over dark surfaces, 0.10 over white/light PDF content).
Design Note — Label Customization The label prop accepts arbitrary strings but the SVG tile width (220px) was sized for "RESTRICTED" (9 chars). Shorter labels look sparse; longer ones clip. A dynamic SVG viewBox or text measurement approach would make the watermark more robust.

§12 — Design Review Annotations

Items for Claude Design review, organized by severity. Items marked Keep should be preserved; Revise are improvement opportunities; Issue are functional problems.

System-level

Keep — Consistent dark base The gray-950 → gray-900 → gray-800 hierarchy creates a clear depth model. Every surface elevation is legible at a glance. Do not introduce additional gray levels or it will dilute the hierarchy.
Keep — Indigo as the sole CTA color Indigo-600/500 is used consistently for every interactive action (button, input focus, badge, score bar, node color, locator text). Resist adding a second accent for edge cases — coherence is a strength here.
Revise — No focus ring on keyboard navigation None of the mockup elements show a focus ring style beyond the indigo input border. Tauri apps are used with keyboards. Add focus-visible:ring-2 focus-visible:ring-indigo-500 focus-visible:ring-offset-2 focus-visible:ring-offset-gray-950 to all interactive elements (buttons, nav tabs, result cards).
Revise — Missing loading skeleton states Graph data and search results both have async fetch paths. Currently there are no skeleton/shimmer states — only the empty states exist. Consider adding a skeleton for the initial load (before data arrives but not "empty").
Issue — No toast/notification system Server save success shows inline text. Auth errors show inline text. Failed publishes would have no visible feedback path. Before production, a lightweight toast component (bottom-right, auto-dismiss 3s) should be considered for transient state feedback.

Auth Shell

Keep — Centered, minimal sign-in The full-screen centered layout is clean and appropriate for an enterprise desktop app. Don't add decorative elements or illustrations — the LogoMark is sufficient.
Revise — Version number positioning absolute bottom-4 means the version overlaps content on small windows. Use a flex column with mt-auto at the bottom of the flex container instead to avoid overlap.
Revise — Error state lacks retry guidance The error message "Authentication failed. Please try again." is generic. Consider surfacing the specific error code from authState.error in a collapsible detail section for enterprise support scenarios.

App Shell & Navigation

Keep — Header height and density 48px header (py-3 + content) is compact and right for a desktop app. Navigation and utility buttons share the row without crowding.
Revise — No visual separator between nav area and utility buttons On wide screens, the space between the last nav tab and the settings button is ambiguous. A thin border-l border-gray-800 before the utility group would clarify the structure.
Revise — DocumentViewer width is fixed 640px On 1280px screens this takes 50% of the viewport for the drawer, leaving Search or Graph with only 640px. Consider a resizable panel or relative width (e.g., 40% min 480px max 720px).

Dashboard

Issue — All data is hardcoded The four tiles display mock data. They should either be clearly marked as "Demo mode" or wired to real APIs before the production readiness milestone.
Revise — No tile refresh mechanism SyncStatus shows "3 min ago" but there's no refresh button or auto-update cycle. A visible last-updated timestamp that refreshes periodically would improve trust signals for enterprise users.
Revise — StorageUsage progress bar color At 21% the indigo bar is fine, but at 80%+ users need a warning signal. Consider amber at 75% and red at 90%. This needs to be data-driven once the API is wired.

Search

Keep — ⌘K global shortcut Keyboard-first access to search is a strong UX choice for a power-user tool. Keep the shortcut hint in the placeholder.
Keep — Client-side facet filtering Filtering the existing result set client-side (rather than re-fetching) gives instant feedback. Appropriate for the current data size.
Revise — FacetPanel width (w-56 = 224px) On narrow windows (e.g., 800px app width) the panel takes 28% of the content area. Consider hiding it behind the toggle by default on window width < 900px.
Revise — Snippet term highlighting regex The highlightSnippet() function splits on whitespace — it won't highlight partial matches or stemmed variants. Fine for MVP; flag as improvement for semantic search.
Revise — ScoreBar threshold signaling The indigo score bar doesn't indicate confidence levels. Consider color transitions: green >80%, indigo 50-80%, amber <50%.

Graph

Keep — Canvas 2D rendering approach For graphs with hundreds of nodes, Canvas outperforms SVG. The viewport culling implementation is the right architectural choice.
Revise — No empty state for graph If /api/v1/graph returns 0 nodes, the canvas renders a blank dark rectangle. Add an empty state ("No knowledge graph data yet — start by ingesting documents").
Revise — NodeInspector Provenance is hardcoded "—" The Provenance field always shows "—". Either wire it to real data or remove the field from the inspector until it's available.
Revise — No loading state for graph fetch The graph data is fetched async. During loading, the canvas area is blank. Add a spinner or skeleton overlay.
Issue — No zoom controls in toolbar The "scroll to zoom" hint is the only zoom affordance. Enterprise users may not discover scroll-to-zoom. Add +/− buttons with reset-to-fit alongside the node/edge count.

Settings

Keep — JWT-sourced profile data Reading name/email/role from the ID token avoids a separate API call and keeps the settings page fast. Good pattern.
Revise — No save error state The server URL save path shows a success state but has no visible error if the URL is invalid or unreachable. Add a red error variant alongside the green success state.
Revise — Sign out button color in a settings form context The bg-red-600/80 destructive button sits directly in a settings card without a separator or warning copy. Consider adding a short explanation ("This will clear your local session and require re-authentication") before the button.
