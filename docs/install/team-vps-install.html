SSH into your server ⏱ 1 min

Open a terminal on your Mac/Linux/Windows machine and connect to the VPS.

Connect to your server

Replace your-server-ip with your VPS's public IP address.

Your local terminal
ssh root@your-server-ip
ℹ️
All remaining commands in this section run on the server, not your own computer.
2

Install system prerequisites ⏱ 3 min

Install Python, git, and system packages. Run as root.

Update and install packages

VPS shell (as root)
apt update && apt upgrade -y apt install -y python3.12 python3.12-venv python3-pip git curl
💡
On Ubuntu 22.04, Python 3.12 may not be in the default repos. If apt install python3.12 fails, run: apt install -y python3 python3-venv python3-pip instead — Python 3.11+ is required.

Install uv (fast Python package manager)

VPS shell
curl -LsSf https://astral.sh/uv/install.sh | sh source $HOME/.local/bin/env 2>/dev/null || export PATH="$HOME/.local/bin:$PATH"
3

Clone DepthFusion and configure it ⏱ 5 min

Create a dedicated user, clone the repo, install the package.

Create a service user

Running DepthFusion as a dedicated non-root user is safer. If you prefer to use an existing user, skip to the next card.

VPS shell (as root)
adduser --disabled-password --gecos "" depthfusion # Or use your own username instead of "depthfusion"

Clone the repository and install

Switch to the service user first. Replace depthfusion with your username if you chose a different one.

VPS shell
su - depthfusion cd ~ git clone https://github.com/gregdigittal/depthfusion.git projects/depthfusion cd projects/depthfusion # Create the virtual environment and install python3 -m venv .venv .venv/bin/pip install --upgrade pip .venv/bin/pip install -e ".[vps]"
ℹ️
The [vps] extras install the HTTP server and all server-side dependencies. No MLX or GPU libraries are installed — VPS mode uses CPU-only BM25 recall with an optional remote LLM backend.

Create the configuration file

Still as the depthfusion user. Replace your-api-key-here with your Anthropic API key.

VPS shell
mkdir -p ~/.claude cat > ~/.claude/depthfusion.env <<'EOF' DEPTHFUSION_API_KEY=your-api-key-here DEPTHFUSION_MODE=vps-cpu DEPTHFUSION_HNSW_ENABLED=false DEPTHFUSION_GRAPH_ENABLED=true DEPTHFUSION_RERANKER_ENABLED=true DEPTHFUSION_TIER_AUTOPROMOTE=true DEPTHFUSION_REST_API=true EOF chmod 600 ~/.claude/depthfusion.env
⚠️
The chmod 600 command makes this file readable only by you — not by other users on the server. Don't skip it.

Optional: Enable the multi-provider bridge (OpenRouter)

If you want the bridge tools (delegate to GPT-4o, Gemini, DeepSeek), add your OpenRouter API key. Skip this if you don't need it.

VPS shell
echo "OPENROUTER_API_KEY=your-openrouter-key-here" >> ~/.claude/depthfusion.env
4

Set up Tailscale on the server ⏱ 5 min

Tailscale creates a private network so team members can reach the server without exposing port 7301 to the public internet.

Install Tailscale on the server

Run this as root (type exit to leave the depthfusion user shell first).

VPS shell (as root)
exit # back to root if you were in the depthfusion user shell curl -fsSL https://tailscale.com/install.sh | sh

Connect the server to Tailscale

Replace your-tailscale-auth-key with an auth key from your Tailscale admin panel (login.tailscale.com/admin/settings/keys).

VPS shell (as root)
tailscale up --authkey=your-tailscale-auth-key --advertise-tags=tag:server
ℹ️
Once connected, get the server's Tailscale IP by running: tailscale ip -4. This is the IP you'll give to team members — it looks like 100.x.x.x.
VPS shell
tailscale ip -4 # Copy this IP — you'll put it in the connection command you send to team members

Lock down port 7301 to Tailscale only

This prevents DepthFusion from being reachable from the public internet. Only Tailscale-connected devices can reach it.

VPS shell (as root)
# Install ufw if not present apt install -y ufw # Allow SSH from anywhere (critical — don't remove this) ufw allow 22/tcp # Block port 7301 from the public internet ufw deny 7301 # Enable the firewall ufw --force enable ufw status
🔒
Tailscale traffic bypasses ufw by design — the tailscale0 interface routes traffic before ufw sees it. So blocking port 7301 in ufw blocks the public internet while still allowing Tailscale connections. This is the correct and intended behaviour.
5

Create and start the systemd service ⏱ 5 min

This makes DepthFusion start automatically on boot and restart if it crashes.

Create the service file

Run as root. Replace depthfusion with your chosen username throughout.

VPS shell (as root)
cat > /etc/systemd/system/depthfusion-mcp.service <<'EOF' [Unit] Description=DepthFusion MCP HTTP Server After=network.target [Service] Type=simple User=depthfusion WorkingDirectory=/home/depthfusion/projects/depthfusion EnvironmentFile=/home/depthfusion/.claude/depthfusion.env ExecStart=/home/depthfusion/projects/depthfusion/.venv/bin/python3 -m depthfusion.mcp.http_server Restart=always RestartSec=5 [Install] WantedBy=multi-user.target EOF systemctl daemon-reload systemctl enable depthfusion-mcp systemctl start depthfusion-mcp

Verify the service started

VPS shell
systemctl status depthfusion-mcp --no-pager curl http://localhost:7301/health
✅
The service status should say active (running). The health check should return {"status":"ok"}.
⚠️
If the status shows activating (auto-restart), check the logs: journalctl -u depthfusion-mcp -n 30 --no-pager. Look for any import errors or missing env vars.

Send each team member the connection command with your Tailscale IP filled in.

What to send team members

Message to team members

Install Tailscale: https://tailscale.com/download Sign in with Google or create a free account. Tell me your Tailscale email so I can approve you. 3. Approving team members in Tailscale

When a team member sets up Tailscale, they appear in your Tailscale admin panel as a pending device. You need to approve them before they can reach the server.

  1. Go to login.tailscale.com/admin/machines
  2. Find the team member's device (they'll tell you their device name or email)
  3. Click the three-dot menu → Approve
💡
You can also enable auto-approval for your Tailscale network to skip manual approval. Go to Access Controls in the Tailscale admin panel and set auto-approval for your email domain.
↑

Updating DepthFusion on the server

Run these commands as the service user when a new version is available.

VPS shell (as service user)
cd ~/projects/depthfusion git pull .venv/bin/pip install -e ".[vps]" -q

Then as root:

VPS shell (as root)
systemctl restart depthfusion-mcp systemctl status depthfusion-mcp --no-pager

Admin troubleshooting

Port 7301 address already in use

A previous instance is still running. Find and stop it:

VPS shell
fuser 7301/tcp # shows the PID holding the port kill <PID> # kill it systemctl restart depthfusion-mcp
ModuleNotFoundError on startup

The package isn't installed in the venv. Run as the service user:

VPS shell (as service user)
cd ~/projects/depthfusion .venv/bin/pip install -e ".[vps]" -q systemctl restart depthfusion-mcp
Team member can connect on Tailscale but depthfusion_status fails
  1. Confirm the service is running: systemctl status depthfusion-mcp
  2. From the team member's machine, try: curl http://100.X.X.X:7301/health (use the Tailscale IP). If it responds, the server is reachable.
  3. If it doesn't respond, check that ufw is not blocking Tailscale traffic: ufw status. The tailscale0 interface should be unrestricted.
  4. Make sure the team member restarted Claude Desktop and started a brand new chat.
How do I check who has connected and what they've stored?

Connect to your server with SSH and run:

VPS shell
# Check the journal for recent connections journalctl -u depthfusion-mcp --since "24 hours ago" --no-pager | grep -i "recall\|publish" # Check memory store size du -sh ~/projects/depthfusion/data/ 2>/dev/null || echo "No data directory yet"

3

Connect Claude to the team server ⏱ 2 min

One command links Claude Desktop to the team memory. Your admin will give you the server IP.

Open Terminal (Mac) or PowerShell (Windows)

🍎 Mac
🪟 Windows
💡
Press ⌘ Command + Space, type Terminal, press Enter.
💡
Press the Windows key, type PowerShell, press Enter.

Run the connection command

Your admin will give you a command that looks like this — with the server's Tailscale IP already filled in. Replace 100.X.X.X with the IP your admin gave you.

Terminal / PowerShell
claude mcp add depthfusion http://100.X.X.X:7301/sse

You should see: Added MCP server depthfusion

⚠️
If you see "command not found: claude" — the Claude Code CLI isn't installed. Use the manual method below instead.
📋 Manual method — if "command not found" appeared

Edit a config file directly. This works just as well.

Mac — Step 1. In Finder, press ⌘ Command + Shift + G and paste this, then press Enter:

Finder path
~/Library/Application Support/Claude/

Windows — Step 1. Press Windows key + R, paste this, press Enter:

Run dialog
%APPDATA%\Claude\

Step 2. Open claude_desktop_config.json with TextEdit (Mac) or Notepad (Windows). Create the file if it doesn't exist.

Step 3. Replace the file contents with this — swap in the server IP your admin gave you:

JSON config
{ "mcpServers": { "depthfusion": { "url": "http://100.X.X.X:7301/sse" } } }

Save the file. Restart Claude Desktop.

Restart Claude Desktop

🍎 Mac
🪟 Windows

Click the Claude icon in your menu bar → Quit Claude. Then reopen from Applications or Dock.

Find Claude in the system tray (bottom-right). Right-click → Quit. Then reopen from the Start menu.

✅ Test the connection

Open a new chat in Claude Desktop and type exactly this:

Claude Desktop — new chat
depthfusion_status
🎉
Success: Claude responds with version info and a list of 29 available tools. You're now sharing memory with the team.
⚠️

Team member troubleshooting

Tailscale is grey or says "Not connected"

Mac: Click the Tailscale menu bar icon → Connect. If the icon isn't visible, search for Tailscale in Spotlight (⌘ Space) and open it.

Windows: Check the system tray (click ∧ near the clock). Right-click Tailscale → Connect.

"command not found: claude" or "claude is not recognized"

Claude Code CLI isn't installed. Use the Manual method expandable section in Step 3 above — it takes 3 extra steps but works just as well.

depthfusion_status gives no response or an error
  1. Confirm Tailscale is connected (solid icon, not grey).
  2. Message your admin to confirm the server is running and your device is approved.
  3. Fully quit Claude Desktop (don't just close the window — use Quit from menu bar or tray).
  4. Reopen Claude Desktop and start a brand new chat.
  5. Type depthfusion_status again.
I need to remove DepthFusion from Claude

If you used the command method:

Terminal / PowerShell
claude mcp remove depthfusion -s user

If you used the manual method: Open claude_desktop_config.json and delete the "depthfusion" block. Save and restart Claude Desktop.
