Join the team network

Tailscale creates a private tunnel to the team server. It runs silently in the background.

What is Tailscale?

Think of it like a private Wi-Fi network that works over the internet — only approved team devices can connect. Without it, the team server can't be reached from outside the office. Setup takes about 3 minutes and you do it once.

Download and install Tailscale

⬇ Download Tailscale

Open the downloaded file and install it — same as any other app. Mac users: drag to Applications. Windows users: run the .exe installer.

3

Connect Claude to the team memory

One command links Claude Desktop to the team server.

Open your terminal / command window

🍎 Mac
🪟 Windows
💡
Press ⌘ Command + Space to open Spotlight. Type Terminal and press Enter. A window with a blinking cursor will open — that's normal.
📸 You'll see text ending with a % symbol and a blinking cursor. That means it's ready.
💡
Press the Windows key on your keyboard, type PowerShell, and press Enter. A dark window will open.
📸 You'll see text starting with PS C:\Users\yourname> and a blinking cursor. That means it's ready.

Run this command

Click Copy, then click inside the terminal window and paste — ⌘ Command + V on Mac, or right-click on Windows — then press Enter.

Terminal / PowerShell
claude mcp add depthfusion http://100.112.109.51:7301/sse

You should see: Added MCP server depthfusion

⚠️
If you see "command not found: claude" — Claude Code CLI isn't installed on your machine. Use the manual method below instead.
📋 Manual method — if "command not found" appeared

This edits a settings file directly. It takes 3 extra steps but works the same.

Mac — Step 1. In Finder, press ⌘ Command + Shift + G and paste this path, then press Enter:

Finder path
~/Library/Application Support/Claude/

Windows — Step 1. Press Windows key + R, paste this, and press Enter:

Run dialog
%APPDATA%\Claude\

Step 2. Open claude_desktop_config.json with TextEdit (Mac) or Notepad (Windows). If the file doesn't exist, create a new text file with that exact name.

Step 3. Replace the entire file contents with this and save:

JSON config
{ "mcpServers": { "depthfusion": { "url": "http://100.112.109.51:7301/sse" } } }

Restart Claude Desktop

🍎 Mac
🪟 Windows

Find the Claude icon in your system tray (bottom right, near the clock). Right-click it → Quit. Then reopen Claude from the Start menu.

4

Check it's working

One message confirms the team memory is connected.

✅ Test the connection

Open a new chat in Claude Desktop and type exactly this (don't add any other words):

Claude Desktop — new chat
depthfusion_status
🎉
Success: Claude will respond with version info and a list of 29 available tools. You're now sharing memory with the team.
⚠️
If nothing happens or you get an error: Make sure Tailscale is running (icon is solid, not grey) and Greg has approved your device. Then try closing and reopening Claude Desktop and start a fresh chat.

Having trouble?

The Tailscale icon is grey or says "Not connected"

Mac: Click the Tailscale icon in your menu bar → Connect. If the icon isn't there, open Spotlight (⌘ Space) and search for Tailscale, then open it.

Windows: Look in your system tray (bottom-right). Click the ∧ arrow to find hidden icons, right-click Tailscale → Connect.

If it asks you to log in again, use the same account you used before.

"command not found: claude" or "claude is not recognized"

Claude Code CLI isn't installed. Use the Manual method (the expandable section just above the Restart step). It takes 3 extra steps and works just as well.

depthfusion_status gives an error or no response
  1. Confirm Tailscale is running — icon should be solid, not grey.
  2. Message Greg to confirm the server is online and your device is approved.
  3. Fully quit Claude Desktop (not just close the window — use Quit from menu/tray).
  4. Reopen Claude Desktop and start a brand new chat.
  5. Type depthfusion_status again.
I need to remove DepthFusion from Claude

If you used the command: Open Terminal / PowerShell and run:

Terminal / PowerShell
claude mcp remove depthfusion -s user

If you used the manual method: Open claude_desktop_config.json and delete the "depthfusion" block. Save and restart Claude Desktop.
