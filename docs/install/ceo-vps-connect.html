📋 Manual method — if the command above didn't work

This method edits a config file directly. You'll use it if the claude command wasn't found.

Step 1. Open Finder. Press ⌘ Command + Shift + G and paste this path:

Finder path
~/Library/Application Support/Claude/

Step 2. Open claude_desktop_config.json with TextEdit. If it doesn't exist, create it.

Step 3. Replace the entire file contents with this (keep your VPS address in the url line):

JSON config
{ "mcpServers": { "depthfusion": { "url": "http://YOUR_VPS_IP:7301/sse" } } }

Step 4. Save the file (⌘S). Quit and restart Claude Desktop.

Troubleshooting
I don't see a response when I type depthfusion_status

This usually means Claude Desktop hasn't loaded the new MCP server yet.

  1. Fully quit Claude Desktop (don't just close the window — quit from the menu/tray icon).
  2. Reopen it.
  3. Start a brand new chat (don't use an existing conversation).
  4. Type depthfusion_status again.

If still nothing: confirm the VPS address is entered correctly above and re-run the connection command.

"command not found: claude" on Mac

This means Claude Code CLI isn't installed on your machine. That's fine — use the Manual method (the expandable section inside Step 2 above) which edits a config file directly and doesn't need the CLI.

"claude is not recognized" on Windows

Same as Mac — Claude Code CLI isn't installed. Use the Manual method inside Step 2 for Windows. It takes 2 extra steps but works just as well.

Claude Desktop doesn't have a config file yet

This happens on a fresh Claude Desktop install. Create the file using the Manual method:

  • Mac: Go to ~/Library/Application Support/Claude/ and create claude_desktop_config.json.
  • Windows: Go to %APPDATA%\Claude\ and create claude_desktop_config.json.

Paste the JSON config from the Manual method above, save, and restart Claude Desktop.

I get a connection error when testing

This usually means the VPS is temporarily offline or the address has a typo. Check:

  1. The VPS address at the top of this page — make sure there are no extra spaces.
  2. Message Greg to confirm the server is running.
  3. Wait 5 minutes and try again (the server may be restarting).
I need to remove or reset the connection

To remove DepthFusion from Claude Desktop:

Mac Terminal: claude mcp remove depthfusion -s user

Windows PowerShell: claude mcp remove depthfusion -s user

Manual: Open the config file and delete the "depthfusion" block, then restart Claude Desktop.
