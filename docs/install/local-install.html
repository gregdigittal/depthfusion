Before you begin — check your Mac

Your Mac needs to meet these requirements. All modern Macs on macOS 14+ qualify.

Requirements checklist

  • 🍎
    Apple Silicon chipM1, M2, M3, or M4 (any variant). Intel Macs are not supported for this mode.
  • 🖥
    macOS 13 Ventura or laterClick the Apple menu → About This Mac to check.
  • 🧠
    8 GB unified memory minimum16 GB or more gives you a better model. 32 GB = best quality.
  • 💾
    25 GB free disk spaceFor the AI model weights (~7–20 GB depending on your memory) plus the app itself.
  • 🌐
    Internet connectionOnly needed during setup to download the model. After that, everything runs offline.
ℹ️
Not sure if your Mac has Apple Silicon? Click the Apple menu (top-left corner) → About This Mac. If it says "Apple M1" / "M2" / "M3" / "M4" (any variant), you're good. If it says "Intel Core", use the Windows/Intel Mac mode instead.

Install Claude Desktop ⏱ 3 min

Download and install Claude Desktop

⬇ Download Claude Desktop for Mac

Run the installer ⏱ 10–40 min

One command does everything: Python, the app, the AI model, and the background service.

Open Terminal

Press ⌘ Command + Space, type Terminal, and press Enter. A black or white window with a blinking cursor appears.

ℹ️
Terminal is a built-in Mac app. You just type commands into it and press Enter to run them. Nothing you do here will harm your Mac.

Paste and run this command

Copy the command below by clicking Copy. Then click inside your Terminal window, paste with ⌘ Command + V, and press Enter.

Terminal
curl -fsSL https://raw.githubusercontent.com/gregdigittal/depthfusion/main/scripts/install-mac-mlx.sh | bash
⚠️
The installer will ask you to do a few things:
  • A dialog may appear asking you to install Xcode Command Line Tools — click Install and wait for it to finish, then press Enter in Terminal when asked.
  • Homebrew (a Mac package manager) may install — this is safe and expected.
  • It will ask for your DepthFusion API key (see Step 3 below).
  • It will ask you to choose an AI model — press Enter to accept the recommended option for your RAM.
  • The model download takes 10–40 minutes depending on your internet speed. Leave the Terminal window open.
3

Get your API key ⏱ 2 min

The installer will pause and ask for this. Get it ready before you run the installer, or while it's downloading.

Where to find your API key

Your DepthFusion API key is an Anthropic API key. It is not the same as your Claude Pro or Claude Max subscription.

  1. Go to console.anthropic.com/settings/keys
  2. Click Create Key
  3. Give it any name (e.g. "DepthFusion local")
  4. Copy the key — it starts with sk-ant-
  5. Paste it into the Terminal when the installer asks
⚠️
The installer will refuse a key starting with sk-ant-api03- — that's the Claude Code billing key, not an API key. Make sure you're on the API Keys page, not the billing page.
4

Restart Claude Desktop and verify ⏱ 2 min

The installer registers DepthFusion with Claude Desktop automatically. You just need to restart it.

Restart Claude Desktop

Find the Claude icon in your menu bar (the row of small icons at the top-right of your screen). Click it → Quit Claude. Then reopen Claude from your Applications folder or Dock.

✅ Test that it's working

Open a new chat in Claude Desktop and type exactly this:

Claude Desktop — new chat
depthfusion_status
🎉
Success looks like: Claude responds with version info and a list of available tools. That means DepthFusion is running on your machine and connected to Claude.
⚠️
If it doesn't respond: The MLX model might still be loading. Wait 60 seconds and try again. The model load can take up to 2 minutes after your Mac starts. If it still fails after waiting, check the troubleshooting section below.

Updating DepthFusion

When a new version is available, run two commands in Terminal.

Run these commands in order

Terminal
cd ~/depthfusion && git pull ~/.depthfusion-venv/bin/pip install -e ".[mac-mlx]" -q

Then restart the background service:

Terminal
launchctl unload ~/Library/LaunchAgents/com.depthfusion.rest.plist launchctl load ~/Library/LaunchAgents/com.depthfusion.rest.plist

Mac troubleshooting

"command not found: brew" or Homebrew errors

Homebrew is a package manager that the installer uses. If you see this error:

  1. Close Terminal completely and open a new one.
  2. Run this command to check if Homebrew is installed: which brew
  3. If nothing prints, visit brew.sh and follow their install instructions, then re-run the DepthFusion installer.
The model download is extremely slow or stopped

Model files are large (7–20 GB). On a slow connection, this can take over an hour.

  • Don't close the Terminal window — it will resume if briefly interrupted.
  • If it's been stuck for 30+ minutes with no progress, press Ctrl + C to stop, then re-run the installer — the download will resume from where it stopped.
  • Use a wired ethernet connection if available for faster downloads.
depthfusion_status says "no tools found" or gives an error
  1. Check if the background service is running: open Terminal and run curl http://127.0.0.1:7300/health. If you see {"status":"ok"}, the service is running.
  2. If nothing responds: run launchctl list | grep depthfusion to see if the service loaded. If it's not listed, re-run the installer.
  3. Check logs for errors: tail -30 ~/Library/Logs/depthfusion-rest.log
  4. Try fully quitting Claude Desktop (menu bar → Quit) and reopening it, then start a brand new chat.
The MLX service keeps crashing (memory errors)

This usually means the AI model is too large for your Mac's memory.

  1. Open Terminal and run the installer again: bash ~/depthfusion/scripts/install-mac-mlx.sh
  2. When it asks about the model, choose a smaller option than before.
  3. The installer will swap the model and restart the service.
How do I uninstall DepthFusion?

Run these commands in Terminal:

Terminal
launchctl unload ~/Library/LaunchAgents/com.depthfusion.mlx-server.plist 2>/dev/null launchctl unload ~/Library/LaunchAgents/com.depthfusion.rest.plist 2>/dev/null rm ~/Library/LaunchAgents/com.depthfusion.*.plist rm -rf ~/.depthfusion-venv ~/depthfusion

Then remove it from Claude Desktop by editing ~/Library/Application Support/Claude/claude_desktop_config.json and deleting the "depthfusion" entry.

💡
What you're installing: DepthFusion on Windows uses BM25 keyword memory — everything you tell Claude is saved and recalled intelligently. Unlike the Mac MLX mode, there is no local AI inference; DepthFusion handles the memory layer and Claude handles the thinking. You can optionally enable the multi-provider bridge (OpenRouter) to delegate to GPT-4o, Gemini, or other models.

Windows troubleshooting

"python is not recognized" after installing Python

Python was installed without adding it to the PATH. Fix it:

  1. Open the Python installer again (from your Downloads folder)
  2. Click Modify
  3. On the next screen, tick Add Python to environment variables
  4. Click Install, then close and reopen PowerShell
Execution policy error in PowerShell

Windows may block script execution by default. Run PowerShell as Administrator (right-click the Start menu → Windows PowerShell (Admin)) and run:

PowerShell (Admin)
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Answer Y and press Enter. Then re-run the install command.

depthfusion_status gives no response or an error
  1. Check the DepthFusion service is running. Open PowerShell and run: curl http://localhost:7300/health. If you see {"status":"ok"}, it's running.
  2. If not running, start it: python -m depthfusion.mcp.server (in the depthfusion folder)
  3. Fully quit Claude Desktop (system tray → Quit) and reopen it, then start a new chat.
