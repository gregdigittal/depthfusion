Local Update Guide

Update DepthFusion on Your Mac

Pull the latest code, sync dependencies, restart services. Takes about 2 minutes.

💡
This guide is for the dev repo install at ~/projects/depthfusion using uv. If you used the one-line installer (repo at ~/depthfusion), see the local install guide update section instead.
1

Open Terminal ⏱ 10 sec

Press ⌘ Space, type Terminal, press Enter.

2

Pull and sync ⏱ 30 sec

Two commands — copy and paste them together.

Terminal
cd ~/projects/depthfusion && git pull && uv sync --extra mac-mlx

You'll see git output, then uv installing any new dependencies. When it finishes, you're running the latest code.

ℹ️
uv sync --extra mac-mlx reads pyproject.toml and updates the .venv with the full Mac MLX stack atomically — equivalent to pip install -e ".[mac-mlx]" but faster and locked. Omitting --extra mac-mlx strips MLX, torch, sentence-transformers, and ChromaDB from the environment.
3

Restart the MCP service ⏱ 5 sec

So Claude Desktop picks up the new code.

Restart the REST / MCP server

Terminal
launchctl unload ~/Library/LaunchAgents/com.depthfusion.rest.plist launchctl load ~/Library/LaunchAgents/com.depthfusion.rest.plist
Also running the local MLX model? Restart that too.

Only needed if you're using mac-mlx mode with a local Gemma/Qwen model. Skip if you're in local or vps-cpu mode.

Terminal
launchctl unload ~/Library/LaunchAgents/com.depthfusion.mlx-server.plist launchctl load ~/Library/LaunchAgents/com.depthfusion.mlx-server.plist
4

Restart Claude Desktop ⏱ 15 sec

Claude Desktop caches the MCP connection — a full quit and reopen is required.

Click the Claude icon in your menu bar (top-right of screen) → Quit Claude.

Then reopen Claude from your Applications folder or Dock.

⚠️
Don't just close the window. Claude keeps running in the background when you close the window. You must choose Quit from the menu bar to fully restart it.
✅ Confirm the update worked

Open a brand new chat in Claude Desktop and type:

Claude Desktop — new chat
depthfusion_status
🎉
Claude responds with version info and the list of available tools. You're on the latest build.

Troubleshooting

git pull says "Already up to date" — is anything wrong?

No. You already have the latest code. Still restart the services and Claude Desktop if you think something changed — the uv sync step updates dependencies even if the source code didn't change.

uv: command not found

uv isn't on your PATH in this Terminal session. Try:

Terminal
source $HOME/.local/bin/env 2>/dev/null || export PATH="$HOME/.local/bin:$PATH" cd ~/projects/depthfusion && git pull && uv sync --extra mac-mlx

If that still fails, install uv: curl -LsSf https://astral.sh/uv/install.sh | sh then open a new Terminal window.

launchctl says "Could not find specified service"

The plist file doesn't exist — either the installer wasn't run on this machine, or it used a different path. Check what's loaded:

Terminal
ls ~/Library/LaunchAgents/ | grep depthfusion

If nothing shows up, the service wasn't installed with launchd. In that case the MCP server is started by Claude Desktop directly and you just need to restart Claude Desktop — no launchctl commands needed.

depthfusion_status still shows the old version after update
  1. Make sure you opened a brand new chat — existing chats cache the tool list.
  2. Fully quit Claude Desktop from the menu bar and reopen it.
  3. Confirm the service restarted: launchctl list | grep depthfusion — it should show a PID.
  4. Check the running version directly: curl http://localhost:7300/health — the response includes the version.
Something broke after the update — how do I roll back?

Roll back to the previous commit:

Terminal
cd ~/projects/depthfusion git log --oneline -5 # find the commit you want to go back to git checkout <commit-hash> # replace with the hash from above uv sync --extra mac-mlx

Then restart services and Claude Desktop as in Steps 3 and 4. To return to the latest: git checkout main && uv sync --extra mac-mlx.
