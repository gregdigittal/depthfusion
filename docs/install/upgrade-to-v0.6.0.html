DepthFusion v0.6.0 — Install & Upgrade

Stable · 2026-05-11 · ffda2bd

🔴 Two P0 silent failures fixed in v0.6.0

  • Mode resolution bug (S-86): DEPTHFUSION_MODE=vps-cpu or vps-gpu silently fell through to BM25-only local retrieval — Haiku reranker and Gemma backend never loaded. Every vps-cpu/vps-gpu install was affected.
  • Missing Anthropic SDK (S-87): pip install depthfusion[vps-cpu] did not install anthropic — Haiku silently degraded to NullBackend on every clean install.

If you have been running vps-cpu or vps-gpu on any version before v0.6.0, you were running BM25-only retrieval with no error message. This upgrade restores the full pipeline.

What's new in v0.6.0 (stable)

  • Mode resolution: DEPTHFUSION_MODE=vps-cpu/vps-gpu now actually activates those modes
  • Missing SDK: anthropic>=0.40 added to [vps-cpu] extras — Haiku reranker works on clean installs
  • New extras deps: fastapi>=0.100, uvicorn>=0.23 in [vps-cpu] and [vps-gpu] (required for RLM sidecar)
  • All 18 MCP tools now return typed JSON schemas — invalid calls produce actionable 422 errors
  • Recall explainability: explain: true on depthfusion_recall_relevant returns per-stage scores
  • SQLite metadata cache: warm recall skips unchanged files — significant cold-start improvement
  • Benchmark harness: scripts/benchmark.py with 8-entry goldset fixture
  • SkillForge RLM HTTP sidecar at port 8771 (loopback-only, FastAPI)
  • Vector embedding consistency: Chroma upsert and query now use the same embedding backend
  • 1519 tests passing (was 1430 at v0.6.0a2)

E. New feature: recall explainability (S-92)

✦ New in v0.6.0 — explain=true parameter

Pass explain: true to depthfusion_recall_relevant and each result now includes BM25 score, RRF score, source weight, reranker rank, and (when applicable) vector score. Useful for diagnosing why a specific block ranked where it did.

bash — smoke test recall + explain
python3 -c "
from depthfusion.mcp.server import _tool_recall
import json
result = json.loads(_tool_recall({'query': 'install verification', 'top_k': 3, 'explain': True}))
blocks = result.get('blocks', [])
print(f'blocks returned: {len(blocks)}')
print(f'error:           {result.get(\"error\", \"none\")}')
if blocks:
    explain = blocks[0].get('explain', {})
    print(f'explain keys:    {list(explain.keys())}')
    print(f'rrf_score:       {explain.get(\"rrf_score\", \"absent\")}')
"

Expected: error: none, rrf_score is a numeric value in the explain dict. A blocks count of 0 is fine on a fresh host.
