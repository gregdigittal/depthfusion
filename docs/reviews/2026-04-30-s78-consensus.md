## Disputed Findings (LOW only ‚Äî does not gate)

### `tests/test_router/test_bus_idempotency.py` ‚Äî `patch.object(..., create=True)` usage

- **Claude (R2)**: maintain ‚Äî "the test file's docstring explicitly documents that `_get_context_bus` does not yet exist at module scope, making `create=True` intentional TDD scaffolding required for the red phase." (Note: Claude's R1 finding said remove; Claude's R2 conceded it should stay.)
- **Codex (R2)**: concede to Claude's R1 ‚Äî "now that `_get_context_bus` is a real module-level function, `create=True` is misleading and should be removed for clarity."
- **Initial decision (this report, original commit)**: keep `create=True`, with the rationale that it preserves a forcing-function semantic. **That rationale was incorrect** ‚Äî see correction below.

### Correction (post-merge cubic.dev review, 2026-04-30)

The third-party `cubic ¬∑ AI code reviewer` running on PR #1 flagged the kept-`create=True` decision and was right. The mechanic of `unittest.mock.patch.object` is the opposite of what the kept-rationale claimed:

- **Without** `create=True`: a missing target attribute raises `AttributeError` immediately at the patch site ‚Üí the test fails *loudly* and the regression is caught.
- **With** `create=True`: a missing target attribute is silently fabricated on the mock ‚Üí the test passes for the wrong reason ‚Üí a rename or deletion of `_get_context_bus` could ship undetected.

So `create=True` actively *masks* the regression class it was kept to catch. The Codex R1 finding ("remove it") and the Codex R2 cross-pollination position were both correct.

**Resolution**: `create=True` removed in the follow-up commit. Both `patch.object` calls now read `patch.object(mcp_server, "_get_context_bus", return_value=bus)`. The test's docstring is updated to record the rationale.

**Lesson recorded**: at LOW severity, two-reviewer consensus can still be wrong. A third independent reviewer is a meaningful additional signal, especially when the original decision turned on subtle library semantics. Consider re-litigating LOW disagreements when a third opinion contradicts.
