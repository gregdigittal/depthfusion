## Disputed Findings (LOW only — does not gate)

### `tests/test_router/test_bus_idempotency.py` — `patch.object(..., create=True)` usage

- **Claude (R2)**: maintain — "the test file's docstring explicitly documents that `_get_context_bus` does not yet exist at module scope, making `create=True` intentional TDD scaffolding required for the red phase." (Note: Claude's R1 finding said remove; Claude's R2 conceded it should stay.)
- **Codex (R2)**: concede to Claude's R1 — "now that `_get_context_bus` is a real module-level function, `create=True` is misleading and should be removed for clarity."
- **Status**: unresolved at LOW. Per skill protocol, LOW findings do not gate the commit. Decision: **keep `create=True`** to preserve the forcing-function semantic — if a future refactor renames or removes `_get_context_bus`, the patch will still bind to the correct attribute name and the test will fail loudly rather than silently drift.
