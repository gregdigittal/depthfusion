### Round 1 reconciliation and fixes applied

Consensus reached after Round 1. No disputes. All three findings were clear-cut.

**3 fixes applied before commit:**

1. **TOCTOU in `apply_feedback` (H-1)** — Restructured to hold `self._lock` for the entire operation, including file I/O. The `atomic_frontmatter_rewrite` helper uses an independent `fcntl.LOCK_EX` sidecar lock (different lock domain), so no deadlock is possible. `entry.applied.add(chunk_id)` is now called inline after each successful file rewrite (instead of collecting in a global set and re-acquiring the lock at the end). Eliminates the race: a second concurrent call now waits at `with self._lock` until the first has finished, and then reads an `already_applied` set that already includes the first call's chunks.

2. **Redundant deferred Path imports (M-1)** — Removed `from pathlib import Path` inside `_discoveries_dir()` and `_resolve_discovery_file()`. The module-level import at line 15 is sufficient.

3. **`_RecallStore` alias (M-2)** — Changed `from depthfusion.core.feedback import RecallStore as _RecallStore` → `from depthfusion.core.feedback import RecallStore` in `_tool_recall_impl`. Consistent with `_tool_recall_feedback`.
