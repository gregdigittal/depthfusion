### Fixes applied (this commit)

1. **Wrong skip-gate on concurrent test** — added `@tool_required` decorator to `test_concurrent_partial_updates_serialize`. The test now skips cleanly when the MCP server module import fails, instead of erroring.
2. **Renamed wrong-bucket test** — `test_non_discovery_chunk_skipped_unsupported` → `test_unresolvable_file_skipped_missing` (matches what the test actually verifies). Added explicit `skipped_unsupported == 0` assertion to lock the bucket boundary.
3. **New test for `skipped_unsupported`** — `test_unregistered_chunk_skipped_unsupported` registers chunk `"registered-stem"` then sends feedback for `"unregistered-stem"`; asserts `skipped_unsupported == 1` and `skipped_missing == 0`. Locks in the previously-uncovered bucket boundary.
4. **Tightened eviction assertion** — `test_sweep_on_write_evicts_stale` now requires `skipped_missing == 1 AND skipped_expired == 0`. An implementation that fails to sweep (only does lazy TTL check) returns `skipped_expired == 1` and would now fail this test.
5. **File-content verification in concurrent feedback test** — `test_concurrent_feedback_different_recall_ids` now reads `fA.md` and `fB.md` after the threads join and asserts both salience scalars went from `1.0` to `1.1`. Catches counter-only implementations.
6. **Spec'd reset helper** — `test_response_shape_has_all_buckets` calls `fb_module.RecallStore.reset_singleton()` rather than `mcp_server._RECALL_STORE_INSTANCE = None`. Aligns with spec §3.2 / Plan Task 3.1's `reset_singleton` classmethod.
