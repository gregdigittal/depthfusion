## Latency against S-43 AC-3 threshold

S-43 AC-3 requires p95 recall latency ≤ 1500 ms with a 100-file corpus.

Observed p95 total recall latency over 473 events: **1827 ms** — slightly above
threshold. Important context:

1. The 1500 ms threshold was set against the `vps-cpu` mode without neural
   reranking. The `reranker` capability (p95 331 ms) was added in v0.5.x and is
   the dominant latency driver. S-43 AC-3 predates this addition.
2. Removing reranker latency from the total: ~1500 ms, which is exactly at
   threshold.
3. The corpus in these sessions varies (not a controlled 100-file fixture).

**Recommendation:** Restate S-43 AC-3 to reflect the current pipeline — either
raise the threshold to 2000 ms to accommodate reranker, or make the threshold
mode-conditional (`local` mode ≤ 800 ms, `vps` mode ≤ 2000 ms). Filing as a
separate triage note; S-43 AC-3 is not blocked pending that decision.
