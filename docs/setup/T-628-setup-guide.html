Desktop App Setup Guide

Install missing system libraries and verify the Tauri app compiles cleanly on this server

What this guide does: The desktop app code is already written and complete. Before it can be compiled, this Linux server needs several system libraries installed. Follow each numbered step in order. Every command is written out in full — just copy and paste it. You do not need to know Rust, TypeScript, or Tauri.

⚠ Before you start

You need to be connected to the server via SSH. Open a terminal and run:

ssh gregmorris@176.9.147.206

Copy the entire block below and paste it into the terminal. Press Enter. It will ask for your password — type it and press Enter (nothing will appear on screen while you type, that is normal). This downloads and installs five packages. It takes roughly 1–2 minutes.

Terminal command

libayatana-appindicator3-dev System tray icon support
librsvg2-dev SVG rendering used for app icons

Troubleshooting

Error: pkg-config could not find: gdk-pixbuf-2.0
Step 1 did not complete properly. Run the two commands below, then repeat Step 5.
Fix command 
sudo apt update && sudo apt install -y libwebkit2gtk-4.1-dev libgdk-pixbuf2.0-dev
Error: pkg-config could not find: webkit2gtk-4.1
The package installed but the system library index needs refreshing. Run this, then repeat Step 5.
Fix command 
sudo ldconfig
Error: E: Unable to locate package libwebkit2gtk-4.1-dev
The apt package list is stale or the package name differs on your Ubuntu version. Run the update command first, then try installing just that one package.
Fix command — Step A 
sudo apt update
Fix command — Step B 
sudo apt install -y libwebkit2gtk-4.1-dev
If Step B still fails, send back the exact error line — the package may need a PPA on older Ubuntu versions.
Red error[E0...] lines (Rust compile errors)
These indicate a code problem rather than an environment problem. Do not try to fix them yourself. Copy everything from the first error[ line to the end of the output and send it back.
