MCP Server IT Admin

The MCP server is the brain of DepthFusion — it stores indexed knowledge, handles queries, and enforces access control. Install it once on your Linux server.

1
Install the server

SSH into your server, then run:

# Clone the repository
git clone https://github.com/gregdigittal/depthfusion.git ~/depthfusion
cd ~/depthfusion
git checkout v2.0.0   # use the latest tag from the releases page

# Create a Python virtual environment
python3 -m venv .venv
source .venv/bin/activate

# Install with server extras (vector store, embeddings, etc.)
pip install -e ".[vps-cpu]"
# GPU server (CUDA 12+): pip install -e ".[vps-gpu]"
2
Run first-time setup

This interactive wizard creates your database files, generates the device CA, and writes ~/.depthfusion/config.json.

python3 -m depthfusion.install.install --mode vps-cpu --v2

The wizard will ask for your OIDC details (tenant ID and client ID from Step 3 below). Have those ready.

⚠️ Never set ANTHROPIC_API_KEY for DepthFusion. Always use DEPTHFUSION_API_KEY separately — this prevents Claude Code billing interference.
3
Configure OIDC sign-in (Azure Entra ID)

Go to Azure Portal → Entra ID → App registrations → New registration and fill in:

FieldValue
NameDepthFusion
Supported account typesAccounts in this organizational directory only
Redirect URIdepthfusion://auth/callback — select Mobile and desktop applications

After creating, note the Application (client) ID and Directory (tenant) ID. Then add these delegated API permissions and grant admin consent:

PermissionTypeWhy
openidDelegatedID token
emailDelegatedUser email in token
profileDelegatedUser name claims
offline_accessDelegatedRefresh token

Add these to your ~/.depthfusion/.env:

DEPTHFUSION_OIDC_TENANT_ID="paste-your-tenant-id-here"
DEPTHFUSION_OIDC_CLIENT_ID="paste-your-app-client-id-here"
4
Start the server
sudo systemctl enable depthfusion-mcp
sudo systemctl start depthfusion-mcp
sudo systemctl status depthfusion-mcp  # should show: active (running)
cd ~/depthfusion
source .venv/bin/activate
DEPTHFUSION_MODE=vps-cpu python3 -m depthfusion.mcp.server

Add to ~/.claude/settings.json:

{
  "mcpServers": {
    "depthfusion": {
      "command": "/home/YOU/depthfusion/.venv/bin/python",
      "args": ["-m", "depthfusion.mcp.server"],
      "env": {
        "DEPTHFUSION_MODE": "vps-cpu",
        "DEPTHFUSION_API_KEY": "your-api-key-here"
      }
    }
  }
}

Restart Claude Code, then run /mcp to confirm depthfusion appears.

5
Set up TLS / reverse proxy Production only

The server listens on 127.0.0.1:7300 by default. In production, put a TLS-terminating reverse proxy in front of it. Never expose port 7300 directly to the internet.

Caddy handles Let's Encrypt certificates automatically. Add to your Caddyfile:

depthfusion.yourcompany.com {
    reverse_proxy localhost:7300
}
server {
    listen 443 ssl;
    server_name depthfusion.yourcompany.com;

    ssl_certificate /etc/letsencrypt/live/depthfusion.yourcompany.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/depthfusion.yourcompany.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:7300;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
6
Enroll the first admin user
python3 -m depthfusion.install.enroll --role owner --email admin@yourcompany.com

This prints an enrollment token. The first sign-in from that email address is auto-granted the owner role.

RoleCan do
ownerEverything · manage users · rotate keys
adminRead all records · manage members · run ingestion
memberRead records where they are in the ACL
viewerRead own records only
✓
Verify the server is running
curl http://localhost:7300/health
# Expected: {"status": "ok", "version": "0.4.0", "mode": "vps-cpu"}
✅ Server is running. Desktop users can now install the app and connect to https://depthfusion.yourcompany.com.

Desktop App End User

Install the DepthFusion desktop app on your computer. Your IT admin will provide the server address before you start.

1
Download and install
  1. Download DepthFusion-<version>-macos-universal.dmg from the GitHub releases page.
  2. Open the .dmg file.
  3. Drag DepthFusion into your Applications folder.
  4. If macOS says "cannot be opened because it is from an unidentified developer":
    → Go to System Settings → Privacy & Security → click Open Anyway
  1. Download DepthFusion-<version>-windows-x64-setup.exe.
  2. Run the installer. If Windows SmartScreen appears, click More info → Run anyway.
  3. The app installs to %LOCALAPPDATA%\DepthFusion\.

IT deployment (silent install):

DepthFusion-<version>-windows-x64-setup.exe /S /ServerUrl=https://depthfusion.yourcompany.com
2
First-time setup
  1. Launch DepthFusion.
  2. Enter the server address your admin gave you
    (e.g. https://depthfusion.yourcompany.com or a Tailscale hostname).
  3. Click Sign in with Microsoft.
  4. A browser tab opens — complete the Microsoft login flow.
  5. The browser redirects back to the app automatically.
  6. You're in. Your session is saved in the OS keychain and persists across restarts.
ℹ️ If the sign-in doesn't redirect back to the app, see the Troubleshooting → Sign-in loop section.
✓
You're ready

The status indicator in the bottom-right corner of the app turns green when the server is reachable and your session is active.

✅ Installation complete. If you see a red status indicator, ask your IT admin to check the server is running and your account has been enrolled.
🔨 Build from source (developers only)

Prerequisites: Rust 1.77+ · Node.js 20+ · Tauri CLI v2

# Install Tauri CLI
npm install -g @tauri-apps/cli@^2

# Build and run
cd depthfusion/app
npm install
npm run dev          # hot-reload dev mode
npx tauri build      # production installer

# macOS universal binary (Intel + Apple Silicon)
rustup target add aarch64-apple-darwin x86_64-apple-darwin
npx tauri build --target universal-apple-darwin
🚨 Before any public release: rotate the Tauri updater signing key. The bootstrap key in the repo must be replaced.
npx tauri signer generate -w ~/.tauri/depthfusion-signing.key
Set the output public key in app/src-tauri/tauri.conf.json → plugins.updater.pubkey and store the private key as the TAURI_SIGNING_PRIVATE_KEY GitHub secret.

SharePoint Connector IT Admin

The SharePoint connector reads files from your Microsoft 365 sites and ingests them into DepthFusion's knowledge store. It uses delta queries — only changed files are fetched after the initial crawl.

1
Create an Azure App Registration for Graph API

In Azure Portal → Entra ID → App registrations → New registration (or open your existing DepthFusion app registration):

Step 1a — Add a certificate:

  1. Go to Certificates & secrets → Certificates → Upload certificate
  2. Generate a cert on your server:
openssl req -x509 -newkey rsa:4096 \
    -keyout ~/.depthfusion/graph-client.key \
    -out ~/.depthfusion/graph-client.crt \
    -days 730 -nodes \
    -subj "/CN=DepthFusion-Graph"

# Secure the private key
chmod 600 ~/.depthfusion/graph-client.key

Upload graph-client.crt to the Azure portal. Note the SHA-1 thumbprint shown after upload.

Step 1b — Add API permissions and grant admin consent:

PermissionTypeWhy
Sites.SelectedApplicationRead only sites you explicitly register (preferred)
Files.Read.AllApplicationRead file content and metadata
User.Read.AllApplicationResolve user IDs for access-control mapping

Click Grant admin consent. Requires Azure Global Admin or SharePoint Admin + Entra Admin.

Step 1c — Grant per-site access (required for Sites.Selected):

Run this PowerShell for each site you want to index. Use Azure Cloud Shell if you don't have PnP Online locally:

# Replace the URL and AppId with your values
Connect-PnPOnline -Url https://yourcompany.sharepoint.com/sites/MySite -Interactive
Grant-PnPAzureADAppSitePermission `
    -AppId "<your-app-client-id>" `
    -DisplayName "DepthFusion" `
    -Permissions Read
2
Configure the connector

Add these to ~/.depthfusion/.env on the server:

DEPTHFUSION_SHAREPOINT_TENANT_ID="your-tenant-id"
DEPTHFUSION_SHAREPOINT_CLIENT_ID="your-graph-app-client-id"
DEPTHFUSION_SHAREPOINT_CERT_PATH="/home/depthfusion/.depthfusion/graph-client.key"
DEPTHFUSION_SHAREPOINT_CERT_THUMBPRINT="ABCDEF0123456789..."  # from Azure portal
⚠️ Set chmod 600 ~/.depthfusion/.env to protect these credentials.
3
Register sites for ingestion
# Register a site (find your drive-id in the Graph Explorer or SharePoint admin center)
python3 -m depthfusion.connectors.sharepoint_scope add \
    --site-url "https://yourcompany.sharepoint.com/sites/MySite" \
    --drive-id "b!ABC..." \
    --label "MySite Documents"

# List registered sites
python3 -m depthfusion.connectors.sharepoint_scope list

# Remove a site
python3 -m depthfusion.connectors.sharepoint_scope remove \
    --site-url "https://yourcompany.sharepoint.com/sites/MySite"
4
Run the initial crawl
python3 -m depthfusion.connectors.sharepoint sync --mode initial

This walks all registered drives and downloads every file. Estimated time: 5–10 minutes per 1,000 documents. The connector automatically rate-limits on 429/503 responses.

ℹ️ A lock file at ~/.depthfusion/sharepoint_sync.lock prevents concurrent runs. If the crawl is interrupted, re-run the same command — it resumes where it left off.
5
Schedule incremental sync

After the initial crawl, set up a cron job to fetch only changed files:

# Edit your crontab: crontab -e
0 * * * * /home/depthfusion/depthfusion/.venv/bin/python \
    -m depthfusion.connectors.sharepoint sync --mode incremental \
    >> /var/log/depthfusion-sharepoint.log 2>&1

Or use the built-in scheduler daemon:

python3 -m depthfusion.connectors.sharepoint_scheduler --interval-minutes 60

Check sync status at any time:

python3 -m depthfusion.connectors.sharepoint status

Troubleshooting

Server won't start: "No module named 'depthfusion'"

You're running Python outside the virtualenv. Activate it first:

source ~/depthfusion/.venv/bin/activate
python3 -m depthfusion.mcp.server

Or set PYTHONPATH manually:

PYTHONPATH=~/depthfusion/src python3 -m depthfusion.mcp.server
SharePoint sync fails: "delta token expired" (HTTP 410)

Delta tokens expire after 30 days of no activity. Reset them and run a full re-crawl:

python3 -m depthfusion.connectors.sharepoint sync --mode initial --reset-cursors
SharePoint sync fails: "another sync process is running"

The previous run crashed without releasing the lock file. Break it manually:

python3 -m depthfusion.connectors.sharepoint sync --break-lock

The lock is PID-aware — if the original process is gone, the next sync attempt clears it automatically.

Desktop app shows "server unreachable" (red status dot)
  1. Is the server running? sudo systemctl status depthfusion-mcp
  2. Does the server address in app Settings match the actual host?
  3. If using Tailscale: tailscale status — is your device connected?
  4. Check TLS: curl -v https://depthfusion.yourcompany.com/health
Desktop app: sign-in loop (browser opens but never comes back)

The depthfusion:// URI scheme must be registered at the OS level. This is done by the installer. If it's missing:

Test: open depthfusion://auth/test — the app should open. If not, re-run the installer.

Check HKCU\Software\Classes\depthfusion in the registry editor. If the key is missing, re-run the installer.

"Authorization denied" when calling tools

Your account hasn't been enrolled or lacks the required role. Ask your IT admin to run:

python3 -m depthfusion.install.enroll \
    --email your@company.com \
    --role member
Tests fail after install 
cd ~/depthfusion
PYTHONPATH=./src python3 -m pytest tests/ -x -q \
    --ignore=tests/test_benchmark \
    --ignore=tests/test_benchmarks \
    --ignore=tests/test_integration

All 3,435 tests should pass. If test_pin.py fails with ImportError: _TOOL_FLAGS, ensure you are on a commit at or after 8537142 (v2.0.0+).
